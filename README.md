# Diplom_1

<h2>Юнит-тестирование класса Burger для Stellar Burgers</h2>

<h3>Описание проекта</h3>
Проект представляет собой набор автоматизированных юнит-тестов для класса Burger из сервиса Stellar Burgers.
Тесты покрывают все основные функциональные возможности класса, включая работу с булками, ингредиентами, расчет
стоимости и формирование чека

<h3>Технологии</h3>

1. Язык программирования: Java 11
2. Фреймворк для тестирования: JUnit 4.13.2
3. Библиотека для мокирования: Mockito 5.19.0
4. Сборка проекта: Maven 3.8.1

<h3>Структура проекта</h3>

- `src/test/java/praktikum/BurgerTest.java` - основной тестовый класс с unit-тестами
- `src/test/java/praktikum/BurgerParametrizedTest.java` - класс с параметризованными тестами

<h3>Тестируемые функциональности</h3>

1.**Работа с булкой**

- Установка булки в бургер
- Проверка корректного присвоения булки

2. **Управление ингредиентами**

- Добавление ингредиентов в бургер
- Удаление ингредиентов по индексу
- Перемещение ингредиентов между позициями
- Обработка ошибок при неверных индексах

3. **Расчет стоимости**

- Расчет цены бургера с булкой и ингредиентами
- Расчет цены бургера только с булкой
- Проверка корректности вычислений

4. **Формирование чека**

- Создание чека для бургера с ингредиентами
- Создание чека для бургера без ингредиентов
- Проверка форматирования и содержания чека

<h3>Особенности реализации</h3>

1. **Мокирование:** Использованы Mockito моки для объектов Bun и Ingredient
2. **Разделение тестов:** Обычные тесты и параметризованные тесты вынесены в отдельные классы
3. **Принцип "один тест - одна проверка":** Каждый тест проверяет только одно утверждение
4. **Изоляция тестов:** Каждый тест запускается с чистым экземпляром Burger
5. **Вспомогательные методы:** Созданы методы для переиспользования кода настройки тестовых данных
6. **Параметризация:** Реализованы параметризованные тесты для проверки различных комбинаций ингредиентов

<h3>Тестовые сценарии</h3>

1. Обычные тесты: Проверка базовой функциональности
2. Параметризованные тесты: Проверка различных комбинаций ингредиентов (3 набора данных)
3. Проверка граничных случаев и исключительных ситуаций
4. Верификация вызовов методов мок-объектов

<h3>Запуск тестов</h3>
Выполните команду: `mvn clean test`

<h3>Покрытие кода</h3>
Проект обеспечивает 100% покрытие кода класса Burger, как требуется в задании.
Все методы класса протестированы с использованием моков и параметризации там, где это необходимо.
